let and const

Block-Scoped Variables

ES5

'use strict';
function foo(){
    console.log(i);
    for (var i = 0; i < 1; i++) {
      console.log(i);
    }
    console.log(i);
};
foo();

ES6

'use strict';
function foo(){
    console.log(i);
    for (let i = 0; i < 1; i++) {
      console.log(i);
    }
    console.log(i);
};
foo();

Arrow Function

'use strict';
var array = [1, 2, 3];
// ES5
array.forEach(function(v) {
    console.log(v);
});
// ES6
array.forEach(v => console.log(v));

Enhanced Object Properties

Property Shorthand
'use strict';
var x=1, y=2;
//ES5
var obj = {x: x, y: y};
//ES6
var obj = {x, y};
Computed Property Names
'use strict';
function getRandomNum() {
  return Math.random();
}
//ES6
let obj = {
    foo: "bar",
    [ 'baz' + getRandomNum() ]: 42
};
//ES5
obj['baz' + getRandomNum()] = 42;
Method Properties
'use strict';
//ES5
var obj = {
    sayHelloTo : function(name) {
       console.log('Hello ' +  name);
    }
}
//ES6
let obj = {
    sayHelloTo (name) {
       console.log(`Hello ${name}`);
    }
}

Module bundler

Four Core Concepts

  • Entry

    Tells webpack where to start and follows the graph of dependencies to know what to bundle.

  • Output

    Tells webpack where to bundle your application..

  • Loaders

    Webpack treats every file (.css, .html, .scss, .jpg, etc.) as a module. However, webpack only understands JavaScript. Loaders in webpack transform these files into modules.

  • Plugins

    Since Loaders only execute transforms on a per-file basis, plugins are most commonly used (but not limited to) performing actions and custom functionality on "compilations" or "chunks" of your bundled modules.
